kafka开机自启动配置

把zookeeper和kafka配置成开机自启动，主要是配置服务文件（里边有服务的描述、启动顺序、服务类型、启动指令、停止指令等等），系统服务的配置文件在“/usr/lib/systemd/system/”目录里边：

添加zookeeper服务的配置文件，配置内容如下

vim /usr/lib/systemd/system/zookeeper.service

 添加kafka服务的配置文件，配置内容如下（也可以偷个懒，复制zookeeper的配置文件，简单修改几个地方就行）

vim /usr/lib/systemd/system/kafka.service

注意环境变量中java路径（此处是本机的java的安装目录），然后kafka在keeper后启动

刷新配置文件

systemctl daemon-reload

zookeeper启动，并配置开机启动

#启动服务 systemctl start zookeeper #设置开机启动 sysetmctl enable zookeeper

查看zookeeper的运行状态

systemctl status zookeeper

kafka启动，配置开机启动

#启动kafka服务 systemctl start kafka #设置kafka开机自启动 systemctl enable kafka

查看kafka的运行状态

systemctl status kafka
